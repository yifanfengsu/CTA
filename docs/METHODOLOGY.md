## 1. 证伪主义研究观

**研究的成功标准 = 产出可信结论，不是产出可部署策略。** 证伪与证实同价：

- 否定性结论与肯定性结论**同等证据标准、同等固化**——防死胡同被反复重走，
  也防假死胡同永久封路（封路的证据也可能是假的：demo 时代全部"死胡同"清单
  在 mainnet 上失效重开，出处 `reports/MR5M_postmortem.md` §7/§8）。
- 一个约束盒被**系统探明为空**是完整的技术成果：13 份研究、预注册判定、
  双样本确认，证明"永续 × ≤4h × 零售 taker × 5–22 币"盒内无可立项策略
  （出处 `reports/PROJECT_FINAL_SUMMARY_20260614.md` §1/§6——项目没有可部署
  策略，但方法论、市场结构图、诚实记录三项资产真实存在）。
- 由此推出报告纪律：数据不支持就写不支持；判死如实归因（死因不硬塞最接近的
  预注册桶）；"考虑过但未改动的文档"显式列出（拒绝隐性决策）。

## 2. 市场效率铁律

**edge 与门槛成正比；人人可得的确定性收益只值无风险利率。**

- 实证之一：期现 cash-and-carry 收敛机制 100% 可靠（合约性），但净年化 4.62%
  < 无风险利率——机制可靠 ≠ 有肉（`reports/basis_arbitrage_feasibility_20260615/`）。
- 实证之二：微观结构"回弹"= bid-ask bounce，即你要付的那个价差本身；速度墙
  内的 edge 属于付了基建门槛的人（`reports/order_flow_exhaustion_feasibility_20260628/`）。
- 实证之三：可预测 ≠ 可货币化——RV 聚集双样本极显著，但线性载体上捕波动率
  退化为方向（`research/_closed/crypto_perp/volatility_event/reports/volatility_event_20260613/`）；VRP 的"溢价"剥离方向后
  是尾部风险的公平保费（`vrp/reports/atm_vrp_stageB_*/`）。
- 推论（选题纪律）：说不出"谁在付钱、为什么钱还在"的想法 = data mining，
  在流水线第 1 站死，零成本。

## 3. 死因库（机制层归因清单）

死因写**机制层**（"延续尾部吃成负期望"），不写参数层（"没调好"）。
完整表格 + 逐条实战出处见 `.claude/skills/PIPELINE.md` §4，此处列纲：

**右偏延续**（100ms→4h 全尺度，四种正交构造五次确认）· **成本墙** ·
**速度墙** · **数据墙** · **市场效率定价** · **流动性伪装**（edge 与拿不到它
是同一事实）· **peso/尾部补偿** · **参数尖峰** · **验证周期不可承受** ·
**描线**（已知靶子单样本改善，5 次实证）。

另有反面模式库（陷阱，出处 `PROJECT_FINAL_SUMMARY` §4.8）：条件厚度陷阱
（median 过 ≠ 可行，看 mean——但左偏方向相反：mean 就是尾部，看拆分）、
IC≠可交易 alpha、减法陷阱（过滤看似提纯实砍好信号）、幸存者偏差方向法、
"少交易让指标变好"陷阱、伪装 beta（须检验而非假设）。

## 4. 闭环机制（研究能力的复利）

```
第8站尸检 → 死因库 → 第0站市场结构地图 + 第1站先验 → 下一个想法在零成本站死掉
```

- 实证链：pairs 漏装噪声标定（231 对 ≈ 噪声底 11.55）→ factor_scale 把噪声
  标定装在第一步 → order_flow 开题即带随机时点安慰剂。同一教训三站复利。
- 防线的通用性：双样本 + 机制解释 + 打折 + 正偏分布四件套，连续拦截 5 次
  "看似改进"（4 次 alpha 增强 + 1 次风控变体）——防线针对的是"样本伪迹/削右尾"
  这一**通用失败模式**，与改的是 alpha 还是风控无关
  （`research/_closed/crypto_perp/trend_b2_4h/reports/b2_4h_vol_targeting_20260628/` Q6.1）。
- **结论会过时（demo 教训：结论可以整体作废），死因库、流水线、防线不会。**
  这就是为什么本仓库把方法论固化成 skills 而不只是报告。
