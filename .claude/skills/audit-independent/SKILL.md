# audit-independent — 独立重推审计（增设的第八技能）

> 增设依据：`research/_closed/crypto_perp/trend_b2_4h/reports/b2_4h_pnl_audit_20260628/` 的范式完整成熟（独立 loader +
> 三项结构检验 + 逐分对帐 + 审计/优化边界声明），且属可复用方法学而非单次技巧。

## 0. 流水线位置

**第 7 站之后 / 平行于全流水线**（见 `.claude/skills/PIPELINE.md`）。
上游输入：判活策略的冻结数字（净利/Sharpe/gate 基线）+ 原始数据源（非引擎
中间产物）。
下游产出：审计报告（确认 / 修正 + 对既有判断的影响表）。确认性结论与修正性
结论**同等价值**（提出正确的怀疑并被数据干净否证 = 与发现真问题同等有价值）。

## 1. 何时用

- 判活策略产出"冻结数字"后（净利/回撤/gate 阈值将被长期引用时）。
- 外部审稿/自我怀疑指向某个具体核算科目（如"funding 是否扣全"）。
- 重大否定性结论要固化进 PROJECT_GUIDE 前的复核（同标准）。

**不用会犯什么错（真实案例）**：
- **复跑引擎 = 循环论证**：引擎若有系统性 bug（求和/边界/符号），复跑会原样
  复现同一个 bug 而看不出。B2_4h funding 审计显式不用 `tb.load_funding`，
  从原始 CSV/zip 经审计脚本自有 loader 独立重推，才把"引擎自证"升级为
  "外部核对"（最大单笔差 $5×10⁻⁵ = 引擎 4 位小数舍入地板，强证据）
  （出处：`research/_closed/crypto_perp/trend_b2_4h/reports/b2_4h_pnl_audit_20260628/` 审计方法段 + Q6.4）。
- **审计顺手变优化**：审计中"顺便改进"参数/口径会污染被审计对象——审计只核对、
  只报告差异，策略与 config 原样不动（该报告的边界声明全文）。

## 2. 怎么用

流程（模板见本目录 `audit_checklist.md`）：

1. **定边界**：写死"这是审计不是优化"——只核对、只报告差异；发现问题 →
   报"高估多少、修正后真实值"，修正的是**核算**不是策略。
2. **独立代码路径**：从**原始数据**（CSV/zip + manifest sha256）重建被审计
   科目，禁止 import 被审计引擎的对应模块；审计脚本与被审计代码**分离**
   （独立文件、独立 loader）。
3. **逐笔比对**：审计值 vs 引擎值逐笔差异（max|Δ|、Σ|Δ|），差异须能被解释到
   机器精度/舍入地板级别。
4. **结构性检验（独立于数值匹配，抓数值巧合掩盖的结构 bug）**：
   - **S1 逐周期**：计数一致性（每笔结算数 ≈ 持仓时长/结算周期）——证明
     "每周期都扣，不是每笔扣一次"；
   - **S2 拼接**：边界不重复/不漏算（always-in 翻仓边界的结算归属，区间
     约定如 `(进,出]` 显式写出并验证）；
   - **S3 符号**：方向约定核对（rate>0 时多头付/空头收，逐笔无矛盾）。
5. **对帐冻结数字**：审计重算 net vs 冻结数字，差额报告到分；给出反事实列
   （"若该科目漏扣会虚高多少"）。
6. **影响表**：逐条列既有判断（冻结数字/Sharpe/前向 gate 口径）是否需要修正
   ——确认性结论也逐条写"无需修正"及理由。
7. **审计产物落盘**：逐笔 ledger（jsonl）+ 数据源 manifest（sha256）+
   audit_summary.json。

## 3. 怎么失效

- **复跑引擎当审计**：为什么——同一代码路径的输出自我一致是恒真式，对系统性
  bug 零检出力；独立性是审计的全部牙齿。
- **只做数值匹配、不做结构检验**：为什么——数值可以巧合地对（两个 bug 抵消、
  样本恰好不触发边界），S1/S2/S3 直接检验机制结构，数值巧合骗不过计数/拼接/
  符号三关。
- **审计与被审计代码共享库函数**：为什么——共享的 loader/求和函数若有 bug，
  两边同错，独立性名存实亡；审计脚本重写关键路径（哪怕重复造轮子——这里
  重复造轮子是特性不是缺陷）。
- **审计中修改被审计对象**：为什么——改动后审计结论对应的已不是被引用的那个
  冻结版本，审计失去对象；审计只读。
- **只报修正、不报确认**：为什么——确认性审计（"数字本就是对的"）防止后续
  被同一怀疑反复消耗，与修正性结论同等固化；漏报确认 = 审计结果不可检索。
- **反事实列缺失**：为什么——"若漏扣会虚高 X"给出该科目的重要性量级
  （funding 占 gross ~14%），没有它读者无法判断审计覆盖的科目是否关键。
