# prior-registry — 机制登记与死因入库（第 1 站 + 第 8 站承载工具）

## 0. 流水线位置

承载 PIPELINE 闭环的两端（见 `.claude/skills/PIPELINE.md` §4）：

- **第 1 站（机制登记与先验）**：开题前查询"我要赌的机制，在本仓库历史上是否被
  测过？什么级别的死亡？"——取先验。
- **第 8 站（尸检入库）**：研究结束后按三级分类学登记死因 / 更新基因。
- **第 -1 站（快速死亡测试）**：Stage -1 第一问"机制是否存在"直接查本 registry。

**闭环地位**：第 8 站写入的内容，构成下一次第 0 站地图与第 1 站先验的**输入**。
第 0 站地图的 Level 1 条目就是本 registry 的数据（地图是它的摘要视图）。
本 skill 偏文档 + 数据（`registry/*.yaml`）+ 轻量查询（`registry_query.py`），
不涉及回测/统计计算——它管的是"研究记忆"。

## 1. 何时用

- **开新研究前（第 1 站 / 第 -1 站）**：`query_death(机制关键词)` 查本机制的历史命运。
  - 命中 **Level 1**（机制失败）→ 须走 **Challenge Path**（第 0 站）说明机制如何不同；
    `check_conflict(想法)` 直接判是否冲突并给须回答的问题。
  - 命中 **Level 2**（市场约束）→ 查"当时够不着的具体条件"（`list_reopenable()`），
    判断我的条件（执行/数据/延迟/资本）是否已变化。
  - 命中 **Level 3**（实现失败）→ 该方向**未被正确测过**，按对应 skill 修正后重测，
    不当作机制否决。
- **研究结束时（第 8 站）**：无论判死判活都登记。死亡按三级分类并写明"接下来该
  做什么"；存活更新 / 新建基因条目（`registry/genomes.yaml`）。**登记是研究
  commit 的一部分**（滞后即闭环断裂）。
- **想复用某机制到新市场**：`query_genome(机制名)` 取基因（core/regime/实证）。

**不用会犯什么错（本仓库真实教训）**：
- MR-5m 若当年被误记为"机制失败"（Level 1）而非"实现失败"（Level 3 数据污染），
  后续四个正交方向（时间尺度/横截面/配对/订单流）的**干净重测就不会发生**——
  误分类会永久关闭一个其实没被正确测过的方向。
- pairs 协整漏查"计数类结论须噪声标定"的先验（231 对 ≈ 噪声底 11.55），factor_scale
  把它装在第一步——**先验不入库，教训不复利**。

## 2. 怎么用 —— 三级死因分类学（核心）

**级别决定后续动作**。误分类的代价是不对称的（见 §3），故分级本身要严谨。

### Level 1 · 机制失败
该机制在本市场**压根不存在**（不是被吃掉，是根本没有）。
- **性质**：永久性（除非市场结构变化）。
- **动作**：进第 0 站地图作为 **posterior belief** 条目；新想法与之冲突须过
  **Challenge Path**（答"为什么这次失效"或"我的机制如何不同"）。
- 本仓库实例：右偏延续主导（MR 时间尺度/funding/配对/订单流/flow-vs-price 全死于
  延续 > 回归，100ms→4h 全尺度）；ETH ATM VRP 无溢价（IV≈RV）；vol-targeting 施于
  正偏策略是范畴错误（降回撤≡削右尾，永久结构真相）。

### Level 2 · 市场约束失败
机制存在，但被市场约束（**成本 / 速度 / 数据 / 流动性 / 效率定价**）吃掉。
- **性质**：条件性——墙的位置取决于研究者的条件。
- **动作**：**必须记录"够不着的具体条件"**（跨过需要什么：maker 执行 / 付费数据 /
  更低延迟 / 更大资本）；条件变化时可重开。缺 `wall_condition` = 未来无法识别可重开。
- 本仓库实例：breakout（薄于成本，需 maker）；factor_scale（流动性伪装，edge 在
  不可交易小币）；BTC ATM VRP（摩擦吃 85% + whipsaw 尾）；basis（效率定价到无风险
  利率）；cross-exchange / on-chain（速度墙，**未实测/外部仓库 → verified: 待核实**）。

### Level 3 · 实现失败
研究过程**自身**的缺陷（数据污染 / 前视 / 事后改门 / 口径错误）。
- **性质**：可预防。
- **动作**：**必须对应一个 skill 或纪律条款**防止重犯；**不构成对该方向的判决**
  （方向仍待正确重测）。
- 本仓库实例：MR-5m（demo 网关污染，13 月作废 → 数据环境铁律）；前向回填污染
  （起点锚定固定日期而非首次真实运行 → 防回填加固）；V1' 事后改门（程序污点，
  降低 B2_4h 证据等级 → 铁律 A）。

> **分级的意义**：Level 1/2 是关于**市场**的（前者永久、后者条件性），Level 3 是
> 关于**研究者自己**的（可修复）。把 Level 1 当 Level 3 → 无效的"修一修再试"循环；
> 把 Level 3 当 Level 1 → 错误关闭一个未被正确测过的方向。

### 策略基因（genome）记录规范
**记机制，不记参数。** 每条基因含：机制名（如 trend-following / behavioral
persistence）、核心机制一句话、盈利来源、风险形态、适用 regime、失效 regime、
实证数字与出处、当前状态。
- **禁止只记参数**：EMA20/100 是**实现**，"延续主导"才是**机制**。记参数无法跨市场
  匹配（回答不了"这个机制在美股/期货是否适用"）；记机制才能在未来研究中复用。
- **已死机制同样登记**：MR 家族=价格回归机制、VRP=卖波动率保险机制……记录"什么
  机制被试过、怎么死的"，它们是第 0 站地图 posterior belief 的数据来源。

### 查询接口（`registry_query.py`，可独立 import）
```
query_death(kw)      # 命中的死亡条目（含 level / next_action），按 level 升序
query_genome(kw)     # 机制基因
check_conflict(m)    # 是否与 Level 1 冲突 → 冲突则返回"须走 Challenge Path"+ 须答问题
list_reopenable()    # 所有 Level 2 条目 + 跨墙所需条件（条件变化时逐条复查）
```
冒烟自检：`python3 registry_query.py`（命中/不误报/冲突判断，当前 19/19 PASS）。

## 3. 怎么失效（每条配"为什么"）

- **分类误置**：为什么——Level 1 当 Level 3 → "修一修再试"的无效循环（机制本就
  不存在，修实现是白费）；Level 3 当 Level 1 → 错误关闭一个未被正确测过的方向
  （MR-5m 若当年记为机制失败，后续四个正交方向的干净重测就不会发生）。分级前先
  问"死因是关于市场还是关于我的研究过程"。
- **基因记参数不记机制**：为什么——参数是市场特定的，机制才可跨市场匹配；记
  EMA20/100 则无法回答"这个机制在美股是否适用"，只能回答"这两个 EMA 值在这段
  历史上如何"——后者对未来研究零复用价值。
- **先验变教条**：为什么——registry 是 posterior belief 不是真理；过强会过滤掉
  机制真正不同的新想法（order-flow exhaustion 若被"回归已死"直接过滤就测不成——
  它赌冲击消散而非价格回归，机制不同故获准测）。与第 0 站 **Challenge Path** 配套：
  冲突不等于禁止，等于"须先说清为什么这次不同"。
- **死因归到参数层**：为什么——"这个参数不行"不是死因，机制层归因才有复用价值；
  参数层归因会导致无穷的参数搜索（"再调调看"），而机制层归因一次性关闭整类
  （趋势线关闭固化的是"Sharpe~0.5 验证周期不可承受"，非"EMA20/100 不好"）。
- **登记滞后**：为什么——研究结束不立即登记 → 闭环断裂，下次重蹈覆辙（先验库的
  价值全在"下一个想法开题前能查到"）；第 8 站应作为研究 commit 的一部分，不是
  "有空再补"。
- **条件性死亡被当永久**：为什么——Level 2 记录"够不着的条件"若缺失，未来条件
  变化时无法识别可重开（maker 执行之于 ATM VRP / breakout，付费数据之于
  factor_scale）；把条件性墙写成永久判决 = 永久丢掉一个本可重开的方向。
