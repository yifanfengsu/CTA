# 第 4 步 — 与"反应式状态熔断器死胡同"的辨析

## (4a) 那次失败的熔断器是什么机制？

来源：`reports/regime/regime_filter_report.md`（任务 A，74 配置全负）。

- **触发信号 = 策略自身近期表现**：A1 = 连续 N 笔止损；A2 = 滚动短窗 PF < 阈值。
- **行为 = 暂停新开仓 N 小时/段**（已有仓位正常出场），暂停后从下一个信号恢复。
- **性质 = 事后反应（lagging）**：等亏损/止损成串之后才暂停。
- **失败根因**：失效月亏损小且分散（占总盈利 ~3%），失效与健康月**共用同一信号源**，
  暂停窗口后常紧跟反弹盈利信号 → 削掉的健康利润 > 省下的失效亏损 → 74/74 配置 Δtot<0。
- 一句话：**"靠少交易让指标变好"(looking better by trading less)** 的陷阱。

## (4b) 本次三条规则是否复现了该失败模式？

判据：**被规则阻止信号的"虚拟 PnL"（误伤额）**。>0 = 拦掉了本会赚钱的（误伤式）；
≤0 = 真的拦了亏的。

| 规则 | 触发信号 | 是否 PnL 驱动 | v2B 误伤额(代表配置) | FLAT 误伤额 | 结论 |
|---|---|:--:|--:|--:|---|
| **R1 日亏熔断** | 当日累计**亏损** | **是（同 A1/A2）** | **+$32,561 ~ +$45,057** | +$19,307↑ | **误伤式，同型** |
| R2 同向上限 | 当前**并发同向持仓数** | 否（结构性） | +$146 (K4) | +$118 (K4) | 非误伤式 |
| R3 横截面降档 | 同 bar **同向信号广度** | 否（结构性） | **−$82** (M4_D1) | −$387 (M4_D1) | 非误伤式（最干净） |

- **R1**：触发信号就是"策略自身的实现亏损"，行为就是"暂停新开仓 N 小时"——与 A1/A2 **机制完全同型**。
  且数据**复现了失败模式**：所有 8 个配置 ΔPnL −11%~−15%，误伤额巨大且为正（v2B +$33k~+$45k）。
  日亏后 MR 常反弹，停新仓恰好砍掉反弹盈利。**这正是反应式熔断陷阱。**
- **R2 / R3**：触发信号是**并发结构**（同向持仓数 / 同 bar 同向信号数），与历史 PnL 无关，是**事前**
  限制相关性暴露而非事后对亏损反应。误伤额接近 0（R3 M4_D1 甚至为负，略拦掉净亏单）。**机制不同型。**

## (4c) 每条规则是否落入"反应式熔断器陷阱"？

- **R1：是。** PnL 触发 + 暂停新仓 + 大额正误伤 + 复现 74 配置的全负模式 → **必须放弃，不进入阶段 2.B。**
- **R2：否。** 结构性触发、低误伤。（但见主报告：它不降 maxDD。）
- **R3：否。** 结构性触发、误伤≈0。（但见主报告：它也不降 maxDD。）
