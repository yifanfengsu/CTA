> 在数据上**不成立**：它能削掉单笔最差，却切不中形成 maxDD 的那个集群亏损下午。

## 第 5 步 — 诚实结论

**(Q1) V1 是否通过双重 gate？** —— **否。15/15 配置全部失败 Gate 2（PnL）。** Gate 1（DD）从来不是约束，
v2B 仅 1.523× 微超，任何降 size 都满足；真正的约束是 PnL，而每个配置都击穿 +50% over FLAT 门槛。

**(Q2) 最佳配置？** —— 无通过配置。按 PnL 排序最不坏的是 **T98_C1**（$251,089，−13.5% vs v2B，仍 +32.7% over FLAT，
DD 1.257×），但**不达 Gate 2**，不构成有效解。

**(Q3) DD 元凶切中率？** —— **12.5–18.8%**。这是 V1 无效的**机制性证据**：cap 没切中要害，maxDD 由中等 ATR 笔的
单日集群亏损制造，而非高 ATR 单笔。cap 的降 DD 纯属"少部署"的副作用。

**(Q4) 4 项诚实性检验？** —— **未执行**（无配置通过 gate，按规范不做；无"最佳配置"可检）。

**(Q5) 综合判断：V1 方向无效，size cap 方向整体终止（路径 A 终止于此）。**

根因（数据驱动，与 CLAUDE.md「数据不支持就说不支持」一致）：
1. **被 cap 的高 ATR 笔 = 利润引擎，不是 DD 来源。** ATR 绝对水平是真 edge（p80-100 PF 2.84、+$21.89/笔，5/5 币）。
   v2B 把它们加注到 LARGE 是**正确的**；任何对它们的截断都直接砍掉 edge 最浓的笔 → PnL 等量塌陷。
2. **代价/收益极度不划算。** Gate 1 只需削 $51 DD（3,426→3,375，1.5%），而最温和的 cap 为此付出 −$39k（−13.5%）PnL。
3. **cap 没切中 DD 元凶**（切中率 12.5–18.8%）。maxDD 是中等 ATR 笔的单日集群，size cap 对它无外科手术能力。
4. **V2/V3 不会救活方向。** 连续衰减（V2）/双轴衰减（V3）只是更温和的 cap——它们丢更少 PnL，但也削更少 DD，
   且无法改变"高 ATR=利润、不是 DD 元凶"这一根本结构。方向问题不是力度问题，**不值得进入 V2**。

**对路径 A 的具体建议（二选一，均不含 size cap）：**
- **(推荐) 接受 v2B 的 1.523× DD。** 它仅微超 1.5×，且源于**收益等比放大**（DD/净利与 FLAT 相同 ~0.0118，风险调整未恶化）。
  应**显式重审"单实现路径上的绝对 1.5× DD"是否为合适门槛**——这是 phase 2a Q4(b)/(c) 已提出的方向。
- **或** 回退 DYN-v1（C2-1）以换更低绝对 DD（但 v2B 净利 +53% > v1 +21%、效率更高、修掉 v1 痛点格，回退代价大）。

> 不做 cherry-pick：虽然 T98_C1 在某些维度"看起来还行"（仍 +32.7% over FLAT），但它**不达 Gate 2**，
> 且 DD 切中率仅 12.5%——它降 DD 靠的是机械少部署，不是精准。按规范**不**把它标记为可用候选。
