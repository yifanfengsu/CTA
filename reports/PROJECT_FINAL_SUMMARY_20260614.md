## §2 项目历程（两个阶段，诚实叙述）

### 阶段一 · MR-5m（约 13 个月）— 最贵的一课

5m fade Donchian 均值回归策略，全套工程落地（vnpy 回测模板 + 独连 OKX 的实盘 runner：IOC 下单 /
仓位对账 / force_close / bills 对账 / 微信通知）。13 个月的优化、调参、模拟盘运行——**全部建立在
OKX DEMO 污染行情上**。2026-06 收尾时发现：(1) 回测库实为 DEMO 撮合行情，含 598 起合成 ramp 异常，
净利与 maxDD 数字双向污染；(2) 在重建的 mainnet 干净数据上重验，MR-5m **无 edge**——全期 PF
0.81–0.85、**毛利 ≈ 0**（$50M+ 成交额上仅 +$6k~$10k 噪声）、净亏 100% 是手续费，DEMO 时代
PF 2.06 中 ~99% 为数据假象。**项目归零。** 完整复盘见 `reports/MR5M_postmortem.md`（含根因分层、
demo 假象机制、§8 新研究启动强制检查清单——本项目阶段二的纪律起点）。

### 阶段二 · 信号空间系统探明（约 3 天）— 把盒子探明

在重建并**双样本验证**的干净数据（OKX mainnet 3.4 年 + Binance 6 年，详见 §5 数据资产）上，
以"前置研究"模式快速、诚实地检验了**方向 / 回归 / 波动率 / 横截面**四大类。13 份研究、零真金
投入、3 天内系统收口。每一份都预注册判定线、跑前写死、死因如实归类、双样本确认。

### 阶段二中的方向震荡（研究心理的真实记录）

这段不粉饰。阶段二并非一条直线：开题先扑向"方向式"的趋势跟踪（trend 六阶段），存活的 B2_4h
因验证周期被关闭后，转向"回归式"（价格 MR 多尺度 / funding 极端回归 / 配对协整）逐一证否，
再试"波动率式"（可预测但永续不可货币化），最后落到"横截面 / 相对价值"。**多次方向跳跃**——
每次一类被否就跳到下一类，一度像在打地鼠。真正的转折是一个认知收敛：这个市场是**右偏延续
结构**，逆势赌方向系统性负期望、顺势弱到验证不起——于是**"赚钱不需要知道涨跌"**（把利润来源
从方向判断移开，走市场中性）成为唯一不被尾部反噬的框架。这个认知让"打地鼠"变成"系统收口"：
不再是随机跳，而是按机制把盒内每一类信号有序划掉。把这段写下来，是因为它对未来的自己有价值
——下次开题时，认知应该从"右偏延续结构 + 验证周期铁律"出发，而不是再从"赌涨跌"出发跳一遍。

## §3 完整的探明地图（项目的技术成果）

### 核心机制结论（置顶 · 项目最持久的认知产出）

**右偏延续结构。** 这个市场（5 币永续，中低频）在**单序列、横截面、价差**三种正交构造下
**三次拒绝逆势**：

- **单序列**：价格 MR（5m / 15m–4h）的回归作为频率现象真实存在，但少数延续尾部移动远大于多数
  小回归，把逆势期望整体吃成负。
- **横截面**：反转因子 F-REV 正 IC 是假象，可交易价差强负（净 −159%，"赢家继续赢"）。
- **价差**：配对协整 |z|≥2 后回归率 17.9% < 无条件 42.4%（z −13.1）；偏离更多是关系断裂的延续，
  价差本身非平稳。

三次构造各自独立，结论同向：**逆势负期望、顺势弱 edge、波动率无载体——同一个右偏延续结构的
三个切面。** 顺势（趋势）收割的正是同一条右偏尾部，所以 B2_4h 能存活；但其 Sharpe ~0.5 在
几年样本下数学上不可能显著。波动率高度可预测，但可预测的是"平静/动荡各自延续"，对线性永续
不可收割（捕波动率必退化为方向）。

## §4 方法论资产（比任何结论都值钱的部分）

这个项目真正的产出不是策略，是一套**被验证有效、可迁移到任何新市场 / 新盒子**的研究方法。逐条：

1. **数据信任优先。** DEMO 污染抹掉 13 个月 → 铁律：唯一可信数据源显式锁定、写库脚本启动打印
   数据环境、新数据必须与外部独立源抽样交叉验证（≥3 随机日逐 bar）才可用。mainnet 库经 Binance
   全量交叉验证（205 月度文件、5 币 1:1 对齐）方才采信。**数据可信是研究的前置条件，不是默认值。**
2. **前置研究模式。** 开题前半天纯描述统计 → 立项算术 → 判定，不写策略、不跑回测、不画 PnL 即
   排除整类方向（第 6–12 研究全部如此）。把"会不会赚钱"的判断从"做完一个策略"提前到"做之前"。
3. **验证周期铁律。** `验证年数 ≈ (1.96 / 年化 Sharpe)²` 作为**选题维度**（topic selection），不是
   事后发现。趋势线在开题日即可知：4h 级、Sharpe~0.5 → 3.4 年永远不够、前向必然以年计。固化进
   CLAUDE.md：验证周期超出可接受范围的方向，无论形态多有吸引力，不立项。
4. **预注册 gate + 不可事后修改 + 死因不强塞。** 阈值跑前写死（1.5× DD gate 差 0.023 不放宽 →
   整条数据事故发现链的起点）；死因如实归类，不硬塞最接近的预注册桶（波动率事件死于增量线而非
   预注册的厚度线，如实修正记录）。
5. **双样本确认（定义样本 + 独立确认样本）。** OKX 描述 + Binance 2020–2026（含 OKX 库不含的 2021
   牛市与 2022 深熊）。识破描线的核心武器——funding 确认那次，OKX 单边全面改善被 Binance 6 年
   完全推翻。
6. **机制验证独立于业绩数字。** 不只看总 Sharpe，直接检验"机制是否成立"（被过滤的信号是否真更差）。
   机制证据在样本间 / 粒度间**翻号**即否，业绩数字再好看也判死（funding 确认的教训）。
7. **已知靶子上单样本改善证据权重 ≈ 0。** 对已有完整历史的资产做增强，能看见答案的样本里的改善
   几乎全是描线；必须独立样本同向确认才采信。
8. **一系列"陷阱"案例（可复用的反面模式库）：**
   - **条件厚度陷阱**：median 厚度过 ≠ 可行，必须看 mean（延续尾部把均值吃成负）。
   - **IC ≠ 可交易 alpha**：因子 IC 显著但 quintile 不单调 / 价差为负 → 不可交易。
   - **减法陷阱**：过滤/确认看似提纯，可能砍掉好信号、频率调整后验证周期反而延长。
   - **幸存者偏差方向**：幸存币样本偏乐观，补退市币只会更差 → 否定性结论更稳。
   - **伪装 beta**：先验担心的"伪装 BTC-beta"经检验被证否（多空两腿 beta 近对称），但"真 alpha"
     仍不足以立项——真 alpha 是必要非充分。
   - **"少交易让指标变好"（looking better by trading less）**：减仓让指标好看是陷阱（MR-5m 熔断器
     74 配置全败、ATR filter 只能少亏的教训）。

**强调：这套方法可迁移到任何新市场 / 新盒子，是项目的可复用资产。盒子换了，流程照用。**

工程资产（阶段一+二建立）：研究引擎链（trend_baseline/validation/r2/portfolio/dualcycle，全部零修改
可复用，含手算自检过的 funding 模块、M2M 逐 bar 记账与逐笔账对账差 $0）；前向 gate 三层设计模板
（FW-KILL/UPGRADE/REVIEW，设计完成未部署，价值在结构）；V1' 集中度质量检验（区分脆弱集中 vs
结构性尾部收割）；MR-5m 实盘执行栈（IOC / force_close / bills 对账，实战验证，归档）。
