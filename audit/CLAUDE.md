# audit/ — 审计域宪法

> 本目录为审计域（核算诚实性/数据取证/结论复核脚本与产物）。宪法先立、脚本后迁
> （重构步骤 2 迁入 `audit_*` / 取证类脚本；本步骤目录暂空）。
> 方法学全文见 `.claude/skills/audit-independent/SKILL.md`；本宪法是其域内约束。

## 1. 独立重推，非复跑引擎

- 审计科目从**原始数据源**（CSV/zip + manifest sha256）经**审计脚本自有代码
  路径**重新推导；**禁止 import 被审计引擎的对应模块**。
- 为什么是铁律：复跑引擎只能复现引擎自身的 bug（循环论证）；独立重推才能抓出
  求和/边界/符号/逐周期错误（范式：`reports/b2_4h_pnl_audit_20260628/`，
  独立 loader 最大单笔差 $5×10⁻⁵ = 舍入地板）。
- 审计中重复造轮子是**特性不是缺陷**。

## 2. 只核对，不修改

- 审计只核对、只报告差异。发现核算错误 → 报"高估/低估多少、修正后真实值、
  对既有判断的影响"；**修正的是核算，不是策略**。
- 被审计对象（引擎/config/冻结数字）在审计期间只读；"顺便改进"= 污染审计对象。
- 绝不触碰：前向系统 / config_frozen / 污染库 / VPS。

## 3. 审计与被审计代码分离

- 审计脚本独立文件、独立 loader、独立求和路径；不共享被审计引擎的关键库函数
  （共享函数同 bug 两边同错，独立性名存实亡）。

## 4. 结构性检验（数值匹配之外的必做件）

数值可以巧合地对（bug 相互抵消/样本不触发边界），故必配结构检验：

- **S1 逐周期**：计数一致性（每笔结算数 ≈ 持仓时长/结算周期）。
- **S2 拼接**：边界不重复/不漏算（翻仓边界结算归属，区间约定显式验证）。
- **S3 符号**：方向约定逐笔无矛盾（如 rate>0 ⇒ 多付/空收）。

分类/口径类审计按同思想设计对应的结构检验（逐周期/拼接/符号是思想模板，
不是穷举清单）。清单模板：`.claude/skills/audit-independent/audit_checklist.md`。

## 5. 确认性结论与修正性结论同等价值

- "数字本就是对的"与"数字虚高 X"同等固化、同等落盘（audit_summary.json +
  逐笔 ledger + 影响表）。
- 提出正确的怀疑并被数据干净否证，与发现真问题同等有价值（demo 惨案后该有的
  核对反射）；不得因结论是确认性的而省略产物。
- 影响表逐条覆盖既有判断（冻结数字/Sharpe/gate 口径），确认也逐条写
  "无需修正"及理由。

## 6. 边界

- 审计域不做研究判定（不判策略生死——那是 research 域第 7 站的事）；
  审计裁的是"数字是否如实"。
- 数据源真实性问题（demo/testnet/污染）移交 data_engineering 域的可证伪核对
  流程；审计域消费其 manifest。
